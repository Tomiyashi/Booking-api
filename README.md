# Room Booking API

Тестовое задание на позицию Backend-разработчика (Go). Сервис для бронирования переговорных комнат с защитой от двойного бронирования и поддержкой слотов.

##  Стек технологий
- **Язык:** Go 1.22+
- **База данных:** PostgreSQL 15+
- **Контейнеризация:** Docker & Docker Compose
- **Драйвер БД:** `lib/pq` (стандартный PostgreSQL драйвер)

##  Архитектурные особенности
- **UUID:** Все сущности (комнаты, слоты, бронирования) используют UUID v4 для безопасности и масштабируемости.
- **Слоты:** Логика построена на фиксированных временных интервалах (слотах), что упрощает выборку свободных окон.
- **Конкурентность:** Защита от Double Booking реализована на уровне БД через `UNIQUE INDEX` с условием `WHERE status = 'active'`. Это гарантирует атомарность бронирования даже при высоких нагрузках.
- **Graceful DB Startup:** В сервисе реализован механизм ретраев (Retry) для подключения к базе данных, что гарантирует стабильный запуск в Docker.

## Запустите проект:

Bash
docker-compose up --build

##  API Эндпоинты
**Общее**
GET /_info — Проверка работоспособности сервиса.

POST /dummyLogin — Получение тестового токена (заглушка).

**Комнаты и слоты**
GET /rooms/list — Получить список всех доступных переговорных комнат.

GET /rooms/{id}/slots?date=YYYY-MM-DD — Получить список свободных слотов в конкретной комнате на выбранную дату.

**Бронирование**
POST /bookings/create — Создать бронирование.

Body: {"slotId": "uuid"}

Returns: 201 Created при успехе, 409 Conflict если слот уже занят.

POST /bookings/{id}/cancel — Отменить активное бронирование.

## Структура проекта
/cmd/api — Точка входа в приложение (main.go).

/internal/server — HTTP-хендлеры и роутинг.

/internal/database — Подключение и настройка PostgreSQL.

/internal/models — Описание структур данных.

/migrations — SQL-миграции для инициализации схемы БД.
